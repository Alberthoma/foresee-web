Cambio 2 — Bug fix: Login bloqueaba la página
🔍 Investigación (código revisado)
Revisé: _syncModalState, handleLogin, onAuthStateChanged, loadUserData, loading-overlay CSS, auth-modal CSS, focus trap keydown listener, handleCalcKeyboard, tab bar event delegation, requestFullscreen, MODAL_IDS, unloadUserData. También privacy-modal y terms-modal como modal-overlays no registradas.
Causa raíz 1 — principal

requestFullscreen() se llamaba dentro de handleLogin() al autenticar con éxito, mientras auth-modal seguía visible (el onAuthStateChanged no había disparado aún). Efectos en fullscreen:

  • ESC → sale del fullscreen (browser-native), no cierra modal → usuario percibe "ESC no funciona"
  • F11 → intenta alternar fullscreen del browser, conflicto con API fullscreen
  • F12 → puede comportarse distinto en modo fullscreen de API
  • El layout cambia abruptamente durante la transición
Causa raíz 2 — secundaria

Sin guard de doble envío. Presionar Enter en el campo password dispara el formulario aunque el botón esté deshabilitado (setAuthLoading(true) deshabilita el botón pero no el event del form). Múltiples llamadas concurrentes a Firebase podían dejar UI inconsistente. Además, setAuthLoading(false) solo se llamaba en el catch — si el login tenía éxito, los botones quedaban disabled (invisible porque el modal se oculta, pero estado corrupto).

Fix 1 — Eliminar requestFullscreen() del login

Removido de handleLogin(). El login es limpio. El usuario puede entrar a fullscreen con el botón "Pantalla Completa" o con el nuevo prompt post-login.

Fix 2 — Guard _authInProgress + finally block
let _authInProgress = false;

async function handleLogin(email, password) {
  if (_authInProgress) return;          // bloquea doble envío
  _authInProgress = true;
  setAuthLoading(true);
  try {
    await signInWithEmailAndPassword(auth, email, password);
  } catch (err) {
    // mensajes de error incluyendo auth/too-many-requests
  } finally {
    _authInProgress = false;          // siempre limpia
    setAuthLoading(false);           // siempre restaura
  }
}
Fix 3 — Mejor mensaje auth/too-many-requests

Antes: "Error al iniciar sesión. Intenta de nuevo." (genérico). Ahora: "Demasiados intentos fallidos. Espera unos minutos e intenta de nuevo."

Fix 4 — privacy-modal y terms-modal en MODAL_IDS

Eran modal-overlay que no estaban en MODAL_IDS. Ahora participan en _syncModalState → body.overflow y aria-hidden se gestionan correctamente cuando se abren.

🔥 Sobre el bloqueo de cuenta Firebase

Firebase Auth bloquea temporalmente por auth/too-many-requests tras ~10 intentos fallidos. Usa backoff exponencial — típicamente unos minutos, puede extenderse si se sigue intentando. El bloqueo se resetea solo. No hay notificación a Firebase Console por defecto (solo en los logs de Auth). La app ahora muestra mensaje claro.

Cambio 3 — Fullscreen post-login restaurado
⚡ Por qué no se puede llamar requestFullscreen() en callbacks async
requestFullscreen() requiere transient user activation — una ventana de ~1 segundo en Chrome desde el último gesto del usuario (clic, tecla). Los callbacks onAuthStateChanged y onSnapshot de Firestore son macrotasks que llegan 1–3 segundos después del gesto. La llamada falla silenciosamente (promesa rechazada sin catch) o lanza "Document is not fully active".
✓
Solución: activar #fullscreen-prompt después de cargar datos
Implementado

El elemento #fullscreen-prompt ya existía en el HTML con logo, mensaje de bienvenida y botón "Iniciar Aplicación" — nunca se había activado. El botón llama requestFullscreen() que es de por sí confiable porque proviene de un clic directo del usuario.

Flujo implementado:

Clic Login → Firebase autentica → onAuthStateChanged → _showFullscreenOnLoad = true → loadUserData → hideLoading() → fullscreen-prompt.active → Clic "Iniciar Aplicación" → ✓ FULLSCREEN 
// onAuthStateChanged
if (user) {
  _showFullscreenOnLoad = true;  // activar flag
  hideAuthModal();
  showLoading();
  loadUserData(user.uid);
}

// En onSnapshot preferencias, después de hideLoading():
if (_showFullscreenOnLoad && !document.fullscreenElement) {
  _showFullscreenOnLoad = false;
  document.getElementById('fullscreen-prompt').classList.add('active');
}

Guards:

  • Si ya está en fullscreen (document.fullscreenElement) → no muestra el prompt
  • Si Firestore actualiza preferencias en background → flag ya es false → no reaparece
  • Al cerrar sesión (unloadUserData) → flag se limpia → próximo login muestra prompt de nuevo
